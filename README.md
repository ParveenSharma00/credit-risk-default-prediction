# Credit Risk Default Prediction

End-to-end machine learning pipeline to predict whether a customer will 
default on their credit balance within 12 months of account acquisition.

Built entirely from scratch — data validation, EDA, feature engineering, 
model training, evaluation, and a production-ready inference script.

## How to Run

### 1. Install dependencies
```bash
pip install scikit-learn pandas numpy joblib matplotlib seaborn
```

### 2. Run notebooks in order
Start from Part1 through Part4_10. Each notebook saves outputs 
that the next one loads.

### 3. Score new applicants
```bash
python3 inference.py --input new_applicants.csv --output scores.csv
```

Output columns:
- `account_id` — original identifier
- `default_probability` — predicted probability of default (0–1)
- `high_risk_flag` — 1 if above threshold, 0 if not
- `risk_band` — LOW / MEDIUM / HIGH / VERY HIGH

## What's Inside

### Part 1 — Data Validation
- 10 integrity checks: leakage, duplicates, merge alignment, label consistency
- Key finding: NaN in credit bureau data means an event never occurred — 
  not missing data. Informative absence.

### Part 2 — Exploratory Data Analysis
- Standalone AUC computed for all 98 numeric features
- Feature family ranking: utilization > repayment history > recency > balance
- Missingness confirmed as an independent predictive signal in both directions
- Bureau score deep-dive with Gini, decile analysis, and capture curve

### Part 3 — Feature Engineering
- 79 binary missingness flags added (NaN encodes credit signal)
- Median imputation fitted on training data only — no leakage
- One-hot encoding for categorical features
- Sample weights (16.28×) for class imbalance correction (16.3:1 ratio)
- Final matrix: 198 features, zero NaN

### Part 4 — Modelling
- Logistic Regression, Random Forest, Gradient Boosting
- All evaluated on 5-fold stratified CV + held-out test set
- Metrics: ROC-AUC, Average Precision, KS Statistic, Gini Coefficient
- 3-tier model selection framework: must-haves → test AUC → CV AUC tiebreaker
- Deep-dive on winner: threshold analysis, calibration, default capture curve, 
  feature importance, error analysis

## Key Technical Decisions

**Why missingness flags?**
In credit bureau data, a missing value typically means an event never occurred
(e.g. no late payment = recency feature is NaN). This absence is itself 
predictive. Adding binary flags preserved this signal rather than throwing 
it away through imputation alone.

**Why sample weights over SMOTE?**
Sample weights rebalance the loss function without altering the data 
distribution. SMOTE generates synthetic minority samples which can 
introduce noise — especially on noisy, anonymised bureau features.

**Why Logistic Regression won?**
With strong L2 regularisation (C=0.05) on 198 features with intentional noise,
LR's linear decision boundary generalised better than the tree models on this 
particular dataset. The signed coefficients are also directly interpretable.

**Evaluation metrics chosen over accuracy**
A naive all-zero classifier achieves 94.2% accuracy on this dataset while 
catching zero defaults. AUC, KS, and Gini are threshold-independent and 
appropriate for imbalanced credit risk data.

## Inference Script

`inference.py` is a standalone production-ready scoring script:
- Loads `best_model.pkl` and `inference_config.pkl`
- Validates input schema before scoring
- Applies full preprocessing pipeline (missingness flags → median imputation 
  → one-hot encoding → column alignment)
- Handles missing columns and unseen categories gracefully
- Outputs probability, binary flag, and risk band per applicant
